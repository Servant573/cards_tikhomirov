Тестовое задание "Карточки"

Описание:
Реализовать сервис отображающий произвольные данные в виде карточек.

Необходимые шаги
1. Сгенерировать произвольные данные в формате json в сервисе https://www.mockaroo.com/

! Одно из полей должно иметь тип MongoDB ObjectID - т.е. идентификатор объекта
! Одно из полей должно иметь тип Avatar.


2. Создать публичный репозиторий на github.com

! Все необходимые файлы должны быть в этом репозитории.
! Ссылку на данный репозиторий будет необходимо отправить.


3. Написать backend-приложение (NodeJS или PHP), которое будет отдавать данные frontend-приложению.

Необходимо реализовать следующие маршруты, отдающие данные из json из пункта 1:
'/?page=1..n' - отдаются краткие данные(необходимые для отображения карточек,
например: фото и название) по текущей странице, если параметр page не указан,
то отдаются данные с первой страницы.
Размер страницы зафиксировать жестко, например 9 или 12.

'/get/:id/' - отдаются полные данные(необходимые для отображения детальной карточки по переданному id объекта,
например: фото, название, профессия, навыки и т.д.)


4. Написать frontend-приложение на VueJS или React,
которое будет отображать список элементов выбранной страницы и детальную карточку.
Внешнее оформление - на Ваше усмотрение.

Необходимые элементы страницы со списком:
Область списка(ограниченное количество кратких карточек текущей страницы, например как в интернет магазинах)
Переключатель страниц(должны выводиться только существующие страницы)

Необходимые элементы детальной страницы:
Должны быть выведены все данные, передающиеся с backend'а.
Должен быть элемент, позволяющий вернуться к списку карточке.


5. Отправить ссылку на репозиторий по адресу sergey.strakhov@arealidea.ru с темой: Выполненное ТЗ (Фамилия Имя)

Опциональные шаги, влияющие на оценку, но необязательные к выполнению.
Ключевые изменения должны быть отражены в гит, а не только пуш последнего результата.
Реализовать поиск(поле поиска на фронтенде и реализация поиска на бэкенде)
Реализовать возможность изменения размера страницы(переключатель на фронтенде и реализация на бэкенде)
